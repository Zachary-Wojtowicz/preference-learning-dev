# CLAUDE.md — Preference Elicitation Experiment

## Overview

A web-based experiment for preference elicitation research. Participants make binary choices between options and optionally adjust dimension sliders. The experiment supports four configurable conditions.

## Files

```
web-interface/
├── index.html              # Single-file experiment app (HTML/CSS/JS)
├── trials.json             # Trial data (generated by generate_trials.py)
├── experiment_config.json  # Condition settings and dimension metadata
├── generate_trials.py      # Script to produce trials from pipeline outputs
└── CLAUDE.md               # This file
```

## Running

```bash
cd web-interface/
python3 -m http.server 8080
# Open http://localhost:8080
```

## Generating Trials from Pipeline Outputs

After the pipeline has produced embeddings, dimensions, and directions:

```bash
python generate_trials.py \
  --config ../method_llm_examples/configs/movies_100.json \
  --embeddings-parquet ../datasets/movies_100/movies_100-embedded.parquet \
  --dimensions ../method_llm_gen/outputs/movies_100/dimensions.json \
  --directions ../method_directions/outputs/movies_100/directions.npz \
  --output-dir . \
  --num-pairs 200 \
  --trials-per-participant 30 \
  --seed 42
```

This produces:
- `trials.json` — all trial data with pre-computed slider projections
- `experiment_config.json` — condition definitions and dimension metadata
- `trial_projections.json` — raw (unscaled) projections for downstream analysis

## Experimental Conditions

The interface supports four between-subjects conditions, controlled by `experiment_config.json`:

| Condition | Choice | Sliders | Adjustable | Free Text |
|-----------|--------|---------|------------|-----------|
| `choice_only` | ✓ | — | — | — |
| `choice_readonly_sliders` | ✓ | ✓ | — | — |
| `choice_adjustable_sliders` | ✓ | ✓ | ✓ | — |
| `choice_sliders_text` | ✓ | ✓ | ✓ | ✓ |

The condition is selected on the setup screen before the experiment begins.

## Data Schema

### Input: trials.json

```json
[
  {
    "trial_id": "t1",
    "option_a_id": "movie_42",
    "option_b_id": "movie_17",
    "prompt": "Which movie would you prefer to watch?",
    "option_a": {
      "label": "The Matrix (1999)",
      "fields": { "title": "The Matrix (1999)", "genres": "Action|Sci-Fi", "director": "Wachowskis", "plot_summary": "..." },
      "text": "fallback plain text (optional)"
    },
    "option_b": { ... },
    "sliders": [
      {
        "id": "dim_1",
        "dimension_id": 1,
        "label": "Action Intensity",
        "low_label": "Not Action-Oriented",
        "high_label": "Highly Action-Oriented",
        "value_if_a": 85,
        "value_if_b": -85
      }
    ]
  }
]
```

Option cards display `fields` as structured key-value pairs when present, falling back to `text` if not. Sliders are dynamically generated from the trial data — any number of sliders per trial is supported.

### Output: response JSON

```json
{
  "participant_id": "P1234",
  "condition": "choice_adjustable_sliders",
  "num_trials": 30,
  "timestamp": "2026-04-21T12:00:00.000Z",
  "responses": [
    {
      "trial_id": "t1",
      "option_a_id": "movie_42",
      "option_b_id": "movie_17",
      "chosen": "a",
      "response_time_ms": 4523,
      "slider_values": { "dim_1": 80, "dim_2": -20, ... },
      "sliders_adjusted": true,
      "free_text": ""
    }
  ]
}
```

Fields present depend on condition:
- `slider_values` and `sliders_adjusted`: only when `show_sliders` is true
- `free_text`: only when `show_free_text` is true
- `response_time_ms`: always present (milliseconds from trial display to submit)

## Randomization

Trial order is deterministically shuffled per participant using a seeded PRNG (seed = hash of participant ID). Each participant sees a random subset of `num_trials_per_participant` trials from the full pool. The same participant ID always produces the same trial sequence.

## Design Notes

- **Single file, no dependencies.** The entire app is one HTML file with inline CSS and JS.
- **No localStorage/sessionStorage.** All state lives in JS variables.
- **Backward compatible.** The old trials.json format (with just `text` on option cards and no `experiment_config.json`) still works — the interface falls back to sensible defaults.
- **Scrolls to top** after each trial submit for long slider panels.
- **Download button** on the results screen lets participants save their response JSON directly.
